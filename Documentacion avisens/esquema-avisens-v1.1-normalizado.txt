// Regla aplicada: cada tabla contiene SOLO los atributos propios
// de la entidad. Las relaciones y los valores calculados NO son
// atributos -> van en tablas intermedias o se calculan por consulta.
//
// 1) Relacion metida como columna -> tabla intermedia
//    - galpones.galpon_asignado_usuario  =>  nueva tabla usuarios_galpones
//
// 2) Listas guardadas como texto -> tabla intermedia
//    - alertas.canales_enviados          =>  nueva tabla alertas_canales
//    - cotizaciones.sensores_incluidos   =>  nueva tabla cotizaciones_sensores
//    - mantenimientos.repuestos_usados   =>  nueva tabla mantenimientos_repuestos
//
// 3) Atributos calculados (derivados) eliminados: se obtienen por
//    consulta o vista, no se almacenan (evita desincronizacion).
//
// 4) EP-03 ampliado (gestion de usuarios): permisos + roles_permisos
//    (control granular), seguridad_cuenta (1:1), sesiones (refresh
//    tokens) y recuperaciones_password. Se devolvieron activo y
//    fecha_creacion a usuarios; intentos_fallidos/bloqueo/ultimo_login
//    pasaron a seguridad_cuenta (no son atributos de la persona).
//
// Total: 39 entidades, 57 relaciones FK.
